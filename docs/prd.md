PRD – 익명 고민 상담 게시판 '마음톡' (Anonymous Advice Board)
==================================================

1. 제품 개요 (Overview)
----------------------
1.1 목적
- 사용자가 완전 무로그인 상태에서 익명으로 고민을 작성하고
  댓글과 좋아요를 통해 공감과 조언을 받을 수 있는 웹 기반 커뮤니티 제공

1.2 핵심 원칙
- 완전 익명 (No Login, No Account)
- 빠른 로딩과 단순한 기능
- 정서적으로 안정적인 UX


2. 타겟 사용자 (Target Users)
----------------------------
- 익명성을 선호하는 일반 사용자
- 실명 기반 SNS가 부담되는 사용자
- 고민, 감정, 일상 문제를 마음 편하게 공유하고 싶은 사용자


3. 인증 및 사용자 식별
---------------------
- 회원가입 없음
- 로그인 없음
- 사용자 식별 방식:
  - 브라우저 기반 임시 식별자 사용
  - Cookie 또는 LocalStorage에 UUID 저장
- 서버는 개인 식별 정보(이름, 이메일, 전화번호 등) 저장 금지


4. 핵심 기능 (Core Features)
----------------------------

4.1 게시글 작성
- 익명 게시글 작성
- 입력 항목
  - 제목 (필수, 최대 50자)
  - 내용 (필수, 최대 1,000자)
  - 카테고리 (선택)
    - 연애 / 진로 / 인간관계 / 학업 / 기타
- 욕설 필터링 적용
  - 욕설 포함 시 작성 불가 또는 자동 마스킹 처리

4.2 게시글 목록
- 기본 정렬: 최신순
- 보조 정렬: 좋아요순
- 목록 표시 정보
  - 제목
  - 좋아요 수
  - 댓글 수
  - 작성 시간 (상대 시간 표시, 예: 2시간 전, 2026.01.01)

4.3 게시글 상세
- 게시글 전체 내용 표시
- 댓글 리스트 표시
- 게시글 좋아요 버튼
- 게시글 삭제 기능
  - 작성자 본인일 경우 언제든 삭제 가능
  - 동일 브라우저 식별자 기준

4.4 댓글 기능
- 익명 댓글 작성
- 대댓글 미지원 (1단 댓글만)
- 댓글 좋아요 가능
- 욕설 필터링 적용
- 댓글 삭제
  - 작성자 본인일 경우 언제든 삭제 가능
  - 시간 제한 없음

4.5 좋아요 기능
- 게시글 / 댓글 좋아요 가능
- 중복 좋아요 방지
  - 동일 브라우저 식별자 기준
- 좋아요 취소 가능

4.6 내가 작성한 글 / 댓글 보기 (My Activity)
--------------------------------------------
- 사용자는 자신이 작성한 게시글을 따로 확인할 수 있어야 한다.
- 로그인 없이 브라우저 기반 식별자를 사용한다.

[내가 쓴 글]
- 동일 브라우저 식별자 기준으로 작성한 게시글 목록 제공
- 표시 정보
  - 제목
  - 작성 시간
  - 좋아요 수
  - 댓글 수
- 게시글 클릭 시 상세 페이지로 이동 가능
- 해당 화면에서 게시글 삭제 가능

[내가 쓴 댓글]
- 선택 사항 (MVP 포함 여부는 자유)
- 동일 브라우저 식별자 기준으로 작성한 댓글 목록 제공
- 댓글 클릭 시 해당 게시글로 이동 가능

[주의사항]
- 브라우저 데이터 삭제 시 (쿠키 / 로컬스토리지 삭제)
  - 작성한 게시글 및 댓글 기록은 복구 불가
- 다른 브라우저 또는 다른 기기에서는 작성 기록 확인 불가

4.7 게시글 검색 기능 (Search)
-----------------------------
- 사용자는 게시글을 검색할 수 있어야 한다.
- 로그인 없이 누구나 사용 가능하다.

[검색 대상]
- 게시글 제목
- 게시글 내용

[검색 방식]
- 키워드 기반 텍스트 검색
- 제목 또는 내용에 키워드가 포함된 게시글 반환
- 대소문자 구분 없음

[검색 UI]
- 게시글 목록 상단에 검색 입력창 제공
- 검색어 입력 후 엔터 또는 검색 버튼 클릭 시 결과 표시
- 검색 결과는 최신순으로 정렬

[제약 사항]
- 댓글 내용은 검색 대상에서 제외
- 복합 조건 검색 (AND / OR) 미지원
- 카테고리 + 검색 조합은 선택 사항

[성능 요구]
- 검색 API 응답 시간은 1초 이내를 목표로 한다


5. 욕설 및 콘텐츠 필터링
-----------------------
- 기본 욕설/비속어 리스트 기반 필터링
- 필터링 대상
  - 게시글 제목
  - 게시글 내용
  - 댓글 내용
- 처리 방식
  - 욕설 포함 시 등록 불가 또는 자동 마스킹 처리


6. UI / UX 디자인 가이드
------------------------
- 전체 테마: 다크 모드 고정
- 배경색: 매우 어두운 색상 (#0F1115 계열)
- 텍스트 색상: 흰색 (#FFFFFF)
- 포인트 컬러
  - 버튼 및 주요 액션: 초록색 계열
- UX 원칙
  - 모바일 우선 (Mobile First)
  - 한 화면 내 핵심 기능 접근 가능
  - 불필요한 애니메이션 최소화


7. 비기능 요구사항 (Non-Functional Requirements)
-------------------------------------------------
7.1 성능
- 초기 로딩 시간: 5초 이내
- API 응답 목표: 1초 이내
- 이미지 업로드 미지원 (텍스트 전용)

7.2 플랫폼
- 반응형 웹 필수
  - Mobile / Tablet / Desktop 대응
- SPA 구조 권장 (구현 방식은 자유)


8. 보안 및 데이터 정책
---------------------
- 개인정보 수집 없음
- IP 주소는 서버 내부 관리 용도로만 사용
- 외부 노출 금지
- 게시글 및 댓글은 영구 저장 (자동 삭제 없음)


9. 기술 스택
------------
- 특정 기술 스택 지정 없음
- AI Coding Agent가 자유롭게 선택
- 단, 성능 요구사항(로딩 5초 이내)은 반드시 충족해야 함


10. MVP 범위
------------
[포함]
- 게시글 CRUD
- 댓글 CRUD
- 좋아요 기능
- 욕설 필터링
- 내가 쓴 글 보기
- 게시글 검색 기능
- 반응형 UI
- 다크 모드

[제외]
- 회원 시스템
- 이미지 업로드
- 알림 기능
- 신고 기능
- 관리자 페이지


11. AI Coding Agent 전달 요약
----------------------------
- 완전 무로그인, 완전 익명 서비스
- 브라우저 기반 식별자로 작성자 구분
- 게시글/댓글 삭제는 언제나 가능
- 내가 쓴 글 목록 제공 필수
- 게시글 제목/내용 검색 기능 필수
- 욕설 필터링 필수
- 반응형 웹 구현
- 초기 로딩 5초 초과 금지
- 기술 스택은 자유 선택
